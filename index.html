Tilt your wrist — move the robot. A real-time wrist-gesture control system that uses the MPU6050 sensor and ESP32 microcontrollers to translate natural wrist movements into precise navigation commands for a wheeled mobile robot.

// 01 — INTRODUCTION

Introduction & Motivation

Imagine guiding a robot simply by tilting your wrist — no joystick, no remote, no dedicated controller. This is exactly what our project delivers. Gesture Controlled Mobile Robot is a wrist-tilt–based human-robot interaction system where an MPU6050 sensor mounted on the wrist detects movement in real time and wirelessly transmits commands to a wheeled mobile robot via ESP32 microcontrollers.

Core Concept: When you tilt your wrist forward, the robot drives forward. Tilt it backward — the robot reverses. Tilt left — robot turns left. Tilt right — robot turns right. No cameras. No special software. Just your natural wrist movements and a pair of ESP32 boards communicating wirelessly.

Traditional mobile robot control uses dedicated hardware — RC remotes, gamepads, or joysticks — all of which require dedicated hardware, add cost, and disconnect human intent from machine action. Our gesture interface collapses this gap entirely. The operator's wrist movement is the controller. This approach has significant real-world impact across multiple domains.

Why This Matters

🏭
Industrial Navigation
Direct robots on factory floors using natural hand tilt, eliminating physical controllers that break or require recharging. Workers already in gloves can gesture-command robots without any additional handheld device.
🔥
Hazardous Environments
In fire, chemical, or nuclear zones, operators in bulky protective suits can use wrist tilt to control inspection robots without holding any external device — intuitive hands-free teleoperation.
♿
Accessibility
Users who cannot operate traditional controllers can use coarse wrist movements to direct assistive robots or delivery platforms — low barrier, high usability.
🎓
Education & Research
A low-cost, open platform simultaneously demonstrating IMU sensing, embedded systems, and wireless communication — ideal for AIE/Robotics coursework projects.

Robot Moves

The MPU6050 on the wrist continuously reads accelerometer and gyroscope data. The transmitter ESP32 processes this sensor data, determines the wrist tilt direction, and encodes a direction command. This command is sent wirelessly to the receiver ESP32 mounted on the robot chassis. The receiver ESP32 drives two DC motors through an L298N H-bridge motor driver, implementing differential-drive locomotion to move the robot in the desired direction.

Key Design Decision: We transmit high-level commands (FORWARD / BACKWARD / LEFT / RIGHT) rather than raw sensor values. This keeps the wireless link simple and ensures the robot responds reliably regardless of link conditions.

// 02 — NOVELTY

What Makes This Novel

Gesture-based robot control has been studied before — but most existing systems are fragmented, camera-dependent, or limited to clunky glove hardware. Our project brings together several well-considered design choices into one cohesive, accessible, hardware-efficient system targeting mobile ground robot navigation.

  • 🤚
    Pure Wrist-Tilt Control — No Camera RequiredUnlike vision-based systems that need a webcam and a processing computer, our system reads directly from the wrist using a compact MPU6050 IMU. The operator simply wears the sensor unit on their wrist — no external setup needed.
  • ↩️
    Intuitive Tilt-to-Direction MappingWrist tilt direction maps directly to robot motion in a way that mirrors natural pointing behaviour. Tilt forward to go forward, tilt back to reverse, tilt sideways to turn — users control the robot correctly on the first attempt with zero training.
  • 📡
    ESP32-to-ESP32 Wireless ArchitectureBoth the transmitter (wrist unit) and the receiver (robot) use ESP32 microcontrollers, enabling reliable wireless communication between them. The same platform handles both sensing and communication, simplifying the overall design.
  • 🔧
    Minimal Component CountThe entire system uses off-the-shelf, low-cost components: MPU6050, two ESP32 boards, L298N motor driver, DC gear motors, and a chassis. No proprietary hardware, no expensive modules — fully reproducible by anyone.
  • ⚡
    Reactive, Real-Time ResponseThe robot follows a reactive paradigm — it responds immediately to wrist tilt changes, similar to human reflex-based motion. There is no buffering or complex inference pipeline, keeping response tight and predictable.
  • 💰
    Ultra-Low Hardware CostESP32 × 2 + MPU6050 + L298N + DC motors + chassis totals well under ₹1500. This makes it one of the most accessible gesture-controlled robot platforms possible to build and replicate.

Summary: A complete, low-cost, end-to-end wrist-gesture navigation pipeline — no cameras, no gloves, no complex software — that proves natural wrist tilt is a viable, intuitive primary control interface for wheeled mobile robots.

// 03 — METHODOLOGY

Methodology

Wrist Gesture Sensing — MPU6050 + ESP32

The MPU6050 is a 6-axis IMU (Inertial Measurement Unit) that combines a 3-axis accelerometer and a 3-axis gyroscope in a single package. It is mounted on the operator's wrist. The transmitter ESP32 reads sensor data over I2C and determines the wrist's current tilt orientation.

⚡ MPU6050 IMU
6-axis sensor providing accelerometer readings (gravity vector) and gyroscope readings (angular rate). The accelerometer axis values directly indicate tilt direction relative to gravity — forward/backward/left/right.
🧠 ESP32 Transmitter
Reads raw sensor data from the MPU6050 over I2C, applies threshold-based logic to determine the dominant tilt direction, and packages it as a direction command for wireless transmission.
📐 Threshold Logic
Accelerometer axis values are compared against calibrated thresholds. The axis with the largest deviation from neutral determines the command: forward, backward, left, or right.

Tilt Detection — Working Principle

When the wrist is held level, the MPU6050 reads gravity evenly across axes. Tilting the wrist shifts the gravity component onto different axes — the transmitter ESP32 maps these axis readings to direction commands using calibrated threshold values.

// Read MPU6050 via I2C on ESP32 transmitter int16_t ax, ay, az, gx, gy, gz; mpu.getMotion6(&ax, &ay, &az, &gx, &gy, &gz); // Determine dominant tilt direction if (ay > THRESHOLD_FWD) sendCommand("FORWARD"); else if (ay < -THRESHOLD_BWD) sendCommand("BACKWARD"); else if (ax < -THRESHOLD_LEFT) sendCommand("LEFT"); else if (ax > THRESHOLD_RIGHT) sendCommand("RIGHT"); else sendCommand("STOP"); // THRESHOLD values calibrated during initial setup

Hardware Components

📦 Wrist Unit
MPU6050 sensor + ESP32 (transmitter). The ESP32 reads the IMU, classifies the tilt, and transmits the command wirelessly to the robot.
🤖 Robot Unit
ESP32 (receiver) + L298N motor driver + 2× DC gear motors + chassis + battery. The receiver ESP32 decodes incoming commands and drives the motors accordingly.

Wrist Tilt → Robot Command Mapping

Raw IMU axis values are decoded into one of four discrete robot commands. The mapping is immediately intuitive — each wrist movement physically implies the corresponding robot motion direction.

Differential Drive Robot Control

The robot uses a two-wheel differential drive configuration with a rear castor for balance. By independently controlling left and right wheel speed, all planar motions are achievable: straight movement, arcs, and point turns.

Hardware Components

🧠 ESP32 Receiver
Receives wireless direction commands from the transmitter ESP32. Decodes the command string and sets PWM + direction signals to the L298N motor driver accordingly.
⚙️ L298N H-Bridge
Dual H-bridge motor driver. Accepts PWM signals from the ESP32 for speed control and direction logic pins to set forward/reverse for each motor channel.
🔧 DC Gear Motors ×2
12V DC gear motors mounted on 65mm rubber wheels. The two motors form the differential drive, enabling forward, reverse, and turning motions.
void handleCommand(String cmd) { if (cmd == "FORWARD") { motorL(150, FWD); motorR(150, FWD); } else if (cmd == "BACKWARD"){ motorL(150, BWD); motorR(150, BWD); } else if (cmd == "LEFT") { motorL(60, FWD); motorR(150, FWD); } else if (cmd == "RIGHT") { motorL(150, FWD); motorR(60, FWD); } else { motorL(0, STOP); motorR(0, STOP); } }

Assembly Status: Robot chassis fully assembled, DC motors mounted and aligned, motor driver interfaced with the receiver ESP32, and basic directional movement verified during testing.

Wireless Communication — ESP32 to ESP32

The transmitter ESP32 (worn on the wrist) communicates wirelessly with the receiver ESP32 (on the robot). Direction command strings are sent over the wireless link whenever the detected tilt direction changes.

📤 Transmitter (Wrist)
The wrist-mounted ESP32 reads the MPU6050, classifies the tilt, and transmits the direction command string to the robot's ESP32 wirelessly.
📥 Receiver (Robot)
The robot's ESP32 listens for incoming command strings. On receiving a command, it immediately updates the motor driver outputs to drive the robot in the indicated direction.
// ESP32 Transmitter — send direction command void sendCommand(String cmd) { // Transmit command string to receiver ESP32 // e.g. "FORWARD", "BACKWARD", "LEFT", "RIGHT", "STOP" wireless.send(cmd); }
// ESP32 Receiver — receive and execute void loop() { if (wireless.available()) { String cmd = wireless.receive(); handleCommand(cmd); // drives motors based on cmd } }

Working Principle: The MPU6050 detects wrist orientation → ESP32 transmitter classifies the tilt and sends the direction → ESP32 receiver drives the L298N motor driver → motors move the robot in the commanded direction.

// 05 — FUTURE IMPLEMENTATIONS

Future Implementations

The current system establishes a solid, working foundation for wrist-gesture–controlled robot navigation. Several practical extensions can be built on top of this platform without significantly increasing complexity — each one adding meaningful capability to the robot.

🚧
Obstacle Avoidance
Adding an ultrasonic sensor (HC-SR04) or IR proximity sensor to the front of the robot would allow it to detect and automatically stop or reroute when an obstacle is detected, even while being gesture-controlled. This is one of the most straightforward and impactful upgrades.
📸
Camera Streaming
Mounting a small camera module on the robot and streaming the feed to the operator's phone or laptop would enable remote navigation in areas the operator cannot directly see — useful for inspection or search tasks.
🔋
Battery Level Indicator
Adding a simple voltage divider circuit with feedback to the operator unit could display the robot's remaining battery level, preventing unexpected shutdowns during operation.
⚡
Speed Control via Tilt Angle
Instead of a fixed PWM speed, the magnitude of the wrist tilt could map to the motor speed — a gentle tilt for slow movement, a deeper tilt for full speed. This would give the operator finer control over robot velocity.
🗺️
Path Recording & Playback
Recording a sequence of gesture commands and replaying them would allow the robot to repeat a mapped path autonomously — useful for routine tasks in structured environments.
🤝
Multi-Axis Tilt Precision
Using both the accelerometer and gyroscope data together with complementary filtering would improve tilt detection accuracy, especially for fine turning manoeuvres at slower speeds.

Key Takeaway: The ESP32 + MPU6050 + L298N architecture is highly extensible. Any of the above features can be added incrementally without redesigning the core system — making this platform well-suited for continued development and research.
